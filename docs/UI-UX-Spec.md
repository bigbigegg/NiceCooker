## 1. 设计系统（Design System）

### 1.1 设计理念

本游戏的设计系统围绕三个核心关键词构建：

- **温暖（Warmth）**：色彩、圆角、阴影共同营造咖啡厅的温暖氛围，让玩家感受到"被治愈"的情感体验。
- **清晰（Clarity）**：作为经营模拟游戏，信息层级必须清晰。玩家需要快速理解资源数量、任务状态、顾客需求，认知负荷需控制在低水平。
- **愉悦（Delight）**：微交互、过渡动画、音效反馈共同构成"令人愉悦"的体验。每次点击都有回应，每次升级都有庆祝。

### 1.3 字体系统

#### 1.3.1 字体家族

```
字体栈（Font Stack）
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  中文显示字体:  "PingFang SC", "Noto Sans SC", "Microsoft YaHei", sans-serif
│  英文/数字字体:  "Nunito", "PingFang SC", sans-serif
│  手写/标题字体:  "ZCOOL XiaoWei", "Ma Shan Zheng", cursive
│  等宽字体:      "JetBrains Mono", "Fira Code", monospace
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

设计理由：
- **Nunito** 是一款圆润的无衬线字体，与卡通可爱风格高度契合。其圆角终端（rounded terminals）让数字和英文在小字号下依然友好可读。
- **PingFang SC** 是 macOS/iOS 系统默认中文字体，渲染质量高，字面大、字间距舒适。
- **ZCOOL XiaoWei / 马善政** 手写字体用于标题和特殊场景（如顾客对话气泡、成就名称），为界面增添手作/手工的温暖质感。
- **等宽字体**仅在数值表格、调试面板等场景使用，确保数字对齐。

#### 1.3.2 字号层级

```
字号系统（Type Scale）
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  h1    28px   页面标题、大弹窗标题                            │
│  h2    24px   面板标题、模态框标题                            │
│  h3    20px   卡片标题、区块标题                              │
│  h4    18px   列表项标题、按钮文字（大）                       │
│  body  16px   正文、按钮文字（中）、表单标签                   │
│  body2 14px   辅助正文、描述文字、列表副标题                   │
│  caption 12px 标注、图例、时间戳、HUD 小字                     │
│  small 10px   极小子 — 角标、快捷键提示、版本号               │
│                                                             │
│  行高:                                                       │
│  h1-h4      1.3  （标题紧凑，增强视觉冲击力）                  │
│  body/body2 1.6  （正文舒适阅读行高）                         │
│  caption    1.4                                                │
│  small      1.2                                                │
│                                                             │
│  字重:                                                       │
│  中文: Regular(400) / Medium(500) / Bold(700)               │
│  英文: Light(300) / Regular(400) / SemiBold(600) / Bold(700) │
│  数字: SemiBold(600) （默认），Bold(700) （强调）             │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

设计理由：
- 字号基于 4px 增量（10/12/14/16/18/20/24/28），与间距系统统一。
- 数字默认使用 SemiBold 字重：等宽数字在游戏界面中频繁出现（金币、数量、等级），SemiBold 确保小字号下的清晰度，同时比 Bold 更柔和。
- 行高 1.6 为中文正文的最优值：中文行高低于 1.5 会产生压迫感，高于 1.8 文字间过于松散。

#### 1.3.3 特殊数字样式

游戏中大量数字（金币、钻石、等级、倒计时）使用**等宽数字（tabular-nums）**特性，确保数字变化时不会产生水平位移。

```
CSS: font-variant-numeric: tabular-nums;
```

### 1.4 间距系统

基于 4px 基础单位，所有间距均为 4 的倍数。4px 在屏幕设计中是"原子间距"——小于 4px 的差异人眼几乎无法分辨，大于 4px 则能形成清晰的空间层次。

```
间距表（Spacing Scale）
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  标记        值        使用场景                               │
│  ────────────────────────────────────────────────────────── │
│  space-0     0px       紧密排列元素间、图标与标签之间          │
│  space-1     4px       相关元素微间距、Badge 内边距           │
│  space-2     8px       同组内元素间距、图标按钮内边距          │
│  space-3     12px      列表项内边距、紧凑卡片内边距           │
│  space-4     16px      标准内边距、组件内间距、按钮内边距      │
│  space-5     20px      卡片内边距、表单项间距                  │
│  space-6     24px      区块间距、面板内边距                   │
│  space-8     32px      大区块间距、弹窗内边距                 │
│  space-10    40px      页面级内边距                           │
│  space-12    48px      页面间大间距                           │
│  space-16    64px      极稀少 — 主要布局分区                  │
│  space-20    80px      页面顶部留白                           │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 1.4.1 组件到组件间距

```
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  同组组件间距: 8px（space-2）                                 │
│  异组组件间距: 16px（space-4）                                │
│  板块间距:     24px（space-6）                                │

```

### 1.5 圆角规范

圆角是卡通/可爱风格的核心视觉元素之一。直角给人严肃、工业的感觉；圆角给人友好、柔软、安全的心理暗示——正如咖啡厅的圆桌和圆杯比尖角家居更让人放松。

```
圆角规范（Border Radius）
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  radius-none    0px     极少使用（表格、分割线）               │
│  radius-sm      4px     内嵌元素、标签、小徽章                 │
│  radius-md      8px     按钮、输入框、列表项                   │
│  radius-lg      12px    卡片、面板                             │
│  radius-xl      16px    弹窗/模态框                            │
│  radius-2xl     24px    大面板、底部弹出面板                   │
│  radius-full    9999px  胶囊形状（按钮、标签、头像）            │
│                                                             │
│  设计理由:                                                    │
│  - 按钮使用 8px 圆角：既不过于方正（破坏可爱感），又不过圆      │
│    （影响可读性）。8px 是安全、友好与清晰之间的平衡点。        │
│  - 卡片使用 12px：比按钮更圆，营造"浮起"的层次感。              │
│  - 弹窗使用 16px：弹窗是最高层级元素，更大的圆角强化"弹出       │
│    层"的视觉暗示。                                            │
│  - 胶囊形（full）用于需要亲和感的元素：店员头像、顾客标签、     │
│    状态指示器。                                                │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 1.6 阴影层级

阴影是 2D UI 中最重要的深度表达手段。本系统定义 4 个层级的阴影，分别对应不同的 Z 轴高度。阴影颜色统一使用棕色（而非黑色），以融入整体温暖色调。

```
阴影层级（Elevation / Box Shadow）
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  Level 0 — 无阴影（Flat）                                    │
│  用于: 平面元素、输入框填充区、表格                            │
│  值:   none                                                 │
│  含义: 元素与背景处于同一平面                                 │
│                                                             │
│  Level 1 — 低浮起（Raised）                                  │
│  用于: 卡片、按钮（默认态）                                   │
│  值:   0px 2px 8px rgba(62,39,35,0.08)                     │
│       0px 1px 2px rgba(62,39,35,0.06)                      │
│  含义: 元素略微高于背景，可交互                               │
│                                                             │
│  Level 2 — 中浮起（Overlay）                                 │
│  用于: 下拉菜单、Tooltip、弹出面板、按钮（Hover态）           │
│  值:   0px 4px 16px rgba(62,39,35,0.10)                    │
│       0px 2px 4px rgba(62,39,35,0.06)                      │
│  含义: 元素明显高于背景，临时展示                             │
│                                                             │
│  Level 3 — 高浮起（Modal）                                   │
│  用于: 模态弹窗、对话框、底部弹出面板                          │
│  值:   0px 8px 32px rgba(62,39,35,0.12)                    │
│       0px 4px 8px rgba(62,39,35,0.08)                      │
│  含义: 最高层级，需要玩家注意并操作                            │
│                                                             │
│  Level 4 — 按下态（Pressed）                                  │
│  用于: 按钮按下、卡片选中                                     │
│  值:   0px 1px 2px rgba(62,39,35,0.06)                     │
│       0px 0px 1px rgba(62,39,35,0.04)                      │
│       内阴影 inset 0px 1px 3px rgba(0,0,0,0.08)            │
│  含义: 元素被"按下去"，凹陷感                                 │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

设计理由：
- 阴影颜色使用 `rgba(62,39,35, ...)` （Primary-900 即最深棕色）而非纯黑。棕色阴影比黑色阴影更自然，模拟了暖光环境下的真实投影色。
- 多层阴影（`0px 4px 16px` + `0px 2px 4px`）产生更自然的投影效果，单层阴影看起来过于"生硬"。
- 按下态使用内阴影（inset）模拟物理按压的凹陷感，这是触觉反馈在视觉层面的唯一替代手段。

### 1.8 动效规范

动效是游戏体验的重要组成部分。在经营模拟游戏中，动效承担三项职能：
1. **提供反馈**：告知玩家操作已生效（按钮按下、金币增加）。
2. **引导注意力**：通过动效将玩家目光引导到重要变化上（顾客进店、成就解锁）。
3. **营造愉悦感**：细腻的动效让操作"有手感"，提升游戏品质感。

#### 1.8.1 过渡时长

```
时长规范（Duration Tokens）
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  即时（Instant）     0ms      无动画（数值跳变、切换状态）      │
│  极快（Micro）       100ms    按钮涟漪、Hover 态切换           │
│  快速（Fast）        200ms    弹窗进出、Tooltip 显隐           │
│  标准（Normal）      300ms    页面切换、卡片展开、列表加载      │
│  缓慢（Slow）        500ms    大面板展开、成就弹窗、场景切换    │
│  极慢（Snail）       800ms    特殊庆祝动画（升级、解锁稀有物品） │
│                                                             │
│  层级关系: 动画影响的范围越大，时长越长。                       │
│  微交互 < 组件动画 < 页面切换 < 场景级动画                     │
│                                                             │
│  设计理由:                                                    │
│  - 100ms 是"人眼能感知但不会觉得等待"的下限，适合 Hover/涟漪    │
│  - 300ms 是 Web 动画的黄金时长：足够流畅，不会拖沓              │
│  - 超过 500ms 的动画仅用于"值得庆祝"的时刻，频繁使用的长动画     │
│    会让操作节奏变慢，影响游戏体验                               │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 1.8.2 缓动函数

```
缓动函数（Easing Functions）
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  ease-out         cubic-bezier(0.00,0.00,0.00,1.00)        │
│  用于: 元素出现（弹窗进入、卡片展开、通知滑入）                 │
│  理由: "快速开始、缓慢结束"，模拟物理世界的减速运动，            │
│        让出现的元素有一种"落定"的感觉。                        │
│                                                             │
│  ease-in          cubic-bezier(1.00,0.00,1.00,1.00)        │
│  用于: 元素消失（弹窗退出、通知滑出）                          │
│  理由: "缓慢开始、快速结束"，让消失的元素"加速离开"，            │
│        减少离开动画对玩家注意力的占用。                        │
│                                                             │
│  ease-in-out      cubic-bezier(0.42,0.00,0.58,1.00)        │
│  用于: 循环动画、状态切换（加载 Spinner、呼吸灯效果）          │
│  理由: 对称的加速与减速，适合无限循环的动画，                    │
│        视觉上不会产生"疲劳感"。                               │
│                                                             │
│  spring           cubic-bezier(0.34,1.56,0.64,1.00)        │
│  用于: 弹跳效果、特殊庆祝动画（成就弹窗、等级提升）            │
│  理由: 超出终点再回弹的过冲（Overshoot）效果，                  │
│        为庆祝类交互增添"弹性/活力/惊喜"的感受。               │
│                                                             │
│  bounce           （使用 GSAP 的 bounce 预设）               │
│  用于: 金币飞入、经验飘字的最终落点                             │
│  理由: 模拟物理弹跳，金币"叮"一声落地，强化获取货币的满足感。    │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

设计理由：
- 弹窗使用 scale 动画（从 0.9 到 1.0）而非滑入：中心弹窗使用缩放 + 淡入最自然，与物理世界"由远及近"的感知一致。
- 确认操作（有破坏性的）的主要按钮放在右侧：符合 F 型阅读模式和右利手用户的触控习惯。
- 模糊遮罩（backdrop-filter）是可选的增强效果：在桌面端开启以增加品质感，在性能较差的移动设备上关闭以保证 60fps。

## 5. 交互设计规范

### 5.1 拖拽操作

拖拽是装修模式和送餐交互的核心操作。需同时支持鼠标（桌面）和触控（移动）。

```
拖拽系统规范
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  触发条件:                                                    │
│    鼠标: mousedown + mousemove（超过 5px 阈值才触发拖拽）     │
│    触控: touchstart + touchmove（同样 5px 阈值）               │
│    阈值理由: 5px 可区分"点击"和"拖拽"，避免误触                │
│                                                             │
│  拖拽中的视觉反馈:                                             │
│    - 被拖对象: 轻微放大（scale 1.05）+ 半透明阴影（Level 3）    │
│    - 原始位置: 虚线边框（2px dashed Primary-300）              │
│    - 有效放置区: 绿色半透明高亮（rgba(76,175,80,0.15)）        │
│    - 无效放置区: 红色半透明高亮（rgba(239,83,80,0.1)）         │
│    - 对齐吸附: 靠近有效网格时自动对齐（6px 吸附距离）           │
│                                                             │
│  拖拽释放:                                                    │
│    有效放置: 对象平滑过渡到目标位置（200ms, spring 缓动）       │
│    无效放置: 对象弹回原始位置（300ms, spring 缓动）            │
│    取消拖拽: 按 ESC 键 / 拖到场景外 → 对象弹回               │
│                                                             │
│  移动端特殊处理:                                               │
│    - 长按家具 300ms 启动拖拽模式（避免与滚动冲突）             │
│    - 拖拽时禁用页面滚动                                       │
│    - 被拖对象跟随手指偏移 40px 向上（避免手指遮挡）            │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 5.5 过渡动画

#### 5.5.1 页面切换动画

```
页面切换规范
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  主页面 → 子页面（深度前进）                                  │
│    子页面从右侧滑入（translateX 100% → 0%）                   │
│    主页面略微缩小并后退（scale 0.95, opacity 0.5）             │
│    时长: 300ms, ease-out                                      │
│    理由: 符合"进入更深层级"的物理空间隐喻。                     │
│                                                             │
│  子页面 → 主页面（返回）                                      │
│    子页面向右滑出（translateX 0% → 100%）                     │
│    主页面恢复（scale 1.0, opacity 1.0）                       │
│    时长: 250ms, ease-in                                       │
│    理由: 返回动画稍快，因为玩家已完成任务，不需要过多等待。     │
│                                                             │
│  Tab 切换（平级切换）                                         │
│    内容区交叉淡入淡出（cross-fade）                            │
│    opacity: 旧的 1→0, 新的 0→1                                │
│    微小的 vertical shift: 新的从下方 8px 升上来                 │
│    时长: 200ms, ease-out                                      │
│    理由: 平级切换不应有方向性动画，交叉淡入是最中性的选择。     │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

#### 5.5.2 列表加载动画

```
列表加载规范
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  骨架屏（Skeleton Screen）                                    │
│    - 用于首次加载和切换分类时的过渡                            │
│    - 骨架元素: 浅灰色圆角矩形，模拟卡片/列表项的轮廓           │
│    - 呼吸动画: 亮度在 90%-100% 之间波动（1.5s 周期）          │
│    - 内容就绪后: 骨架屏淡出 + 真实内容淡入（200ms）            │
│                                                             │
│  交错入场（Staggered Entrance）                               │
│    - 列表/网格项逐个出现                                      │
│    - 每个项目延迟: 30ms（第1个 0ms, 第2个 30ms, 第3个 60ms…） │
│    - 单个项目动画: opacity 0→1, translateY(12px)→0            │
│    - 时长: 250ms per item, ease-out                           │
│    - 最多延迟: 前 15 项（避免列表过长时等待过久）              │
│    理由: 交错入场让列表加载有"节奏感"，比全部同时出现更有品质。 │
```

### 5.6 微交互

微交互是"游戏的灵魂"——它们不改变游戏数值，但决定了操作是否令人愉悦。

```
微交互规范
┌─────────────────────────────────────────────────────────────┐
│                                                             │
│  1. 按钮涟漪（见 5.2 节）                                     │
│                                                             │

│      - 图标从小到大（scale 0→1.2→1.0, spring 缓动）           │
│      - 随后缓缓上升并淡出（translateY -40px, opacity 1→0）     │
│      - 持续 800ms                                             │
│      - 一次可同时飘出 1-3 个                                  │
│    理由: 正向反馈让玩家感受到"被认可"，是强化的关键。           │
│                                                             │
│  3. 金币飞入动画                                              │
│    触发: 出售商品、完成订单、领取成就奖励时                   │
│    效果:                                                     │
│      - 金币图标从产生位置（顾客/设备）飞向顶部状态栏           │
│      - 轨迹: 抛物线（贝塞尔曲线，起点 → 弧形顶点 → 终点）     │

│      - 时长: 600ms, bounce 缓动（终点弹跳）                    │
│      - 到达后: 顶部金币数字轻微放大（scale 1.1）再恢复        │
│      - 数量文字同步递增（odometer roll）                      │
│    理由: 物理化的金币流动让"赚钱"变得具象化、沉浸化。           │
│          抛物线轨迹 + 终点弹跳模拟了真实的抛掷感。              │
│                                                             │
│  4. 经验获取飘字                                              │
│    触发: 任何获得经验的操作                                   │
│    效果:                                                     │
│      - 角色头顶飘出 "+15 EXP" 白色文字                        │
│      - 文字上升 30px 后淡出（600ms）                          │
│      - 字号: 16px, SemiBold, 带 1px 深色阴影确保可读性        │
│    理由: 类似 MMORPG 的经验获取反馈，给玩家即时的成长感。       │
│                                                             │
│  5. 成就解锁弹窗（见 6.5 节）                                 │
│                                                             │
│  6. 开关切换动画                                              │
│    触发: Toggle Switch 被点击                                 │
│    效果:                                                     │
│      - 圆形滑块平滑滑到另一侧（200ms, ease-out）               │
│      - 背景色渐变切换: 关闭灰(#BDBDBD) → 开启绿(#4CAF50)      │
│      - 伴随微弱的"咔嗒"音效（如实现）                         │
│    理由: 物理开关的拟物设计，降低认知成本。                     │
│                                                             │
│  7. 数字跳变（Odometer Roll）                                │
│    触发: 数值变化（金币/钻石/经验/等级）                       │
│    效果:                                                     │
│      - 每位数字独立向上滚动到新数字                             │
│      - 每位延迟: 50ms（个位先变，然后十位、百位...）           │
│      - 每位动画时长: 200ms, ease-out                          │
│    理由: 比直接替换数字更有"积累感"，类似于老虎机/计数器。       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```
