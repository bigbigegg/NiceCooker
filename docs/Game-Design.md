## 1. 核心循环设计

### 1.1 三层目标体系

游戏设计遵循「短期-中期-长期」三层目标体系，确保玩家在任何时间尺度上都有明确的追求方向。

#### 1.1.1 短期目标（单次游戏会话，5-15 分钟）

| 目标 | 描述 | 完成奖励 | 刷新频率 |
|------|------|---------|---------|
| 完成每日任务 | 完成 3-5 个日常经营小目标 | 金币 500-2000 + 钻石 1-3 | 每日（游戏内1天） |
| 服务一波客流 | 完成一个客流高峰时段的接待 | 金币收入 + 经验值 | 每波客流 |
| 提升食谱熟练度 | 制作某食谱 N 次，提升品质稳定性 | 该食谱品质下限提升 | 持续累积 |
| 触发一次随机事件 | 偶遇特殊事件（流浪猫、博主探店等） | 金币/钻石/特殊道具 | 随机 |
| 获得一次小费 | 制作完美咖啡 / 满意度满星 | 额外金币 + 经验 | 即时 |

**节奏设计**：每个短期目标完成间隔约 3-5 分钟，形成密集的正反馈循环。

#### 1.1.2 中期目标（跨天游戏，1-7 天）

| 目标 | 描述 | 完成奖励 | 典型耗时 |
|------|------|---------|---------|
| 店铺升级 | 累积经验升至下一等级 | 解锁新内容 | 2-4 小时游戏时间 |
| 解锁新食谱 | 达到等级/购买设备后解锁 | 新商品可售卖 | 1-3 小时 |
| 购买核心设备 | 积攒金币购买/升级设备 | 提升制作效率/品质 | 30 分钟-2 小时 |
| 雇佣新员工 | 达到等级后解锁招聘 | 自动化经营 | 1-2 小时 |
| 完成一个装修主题 | 收集购买全套主题家具 | 店铺风格统一 | 2-5 天 |
| 培养回头客 | 服务同一顾客 3 次以上 | 顾客成为常客，消费提升 | 2-4 天 |
| 完成成就系列 | 完成某个类别的全部成就 | 钻石 + 限定装饰 | 3-7 天 |

**节奏设计**：中期目标形成「存钱→消费→成长→新目标」的循环，每次消费后立即感受到效率/体验提升。

#### 1.1.3 长期目标（跨周/月游戏，7-30+ 天）

| 目标 | 描述 | 完成奖励 | 典型耗时 |
|------|------|---------|---------|
| 店铺满级（Lv20） | 店铺达到最高等级 | 终极称号 + 店铺外观升级 | 20-30 天 |
| 食谱全收集 | 解锁并制作所有食谱（含隐藏） | 终极成就「咖啡大师」 | 15-25 天 |
| 员工全满级 | 所有员工达到最高等级 | 店铺效率最大化 | 15-20 天 |
| 装修全收集 | 收集所有家具和装饰品 | 自由搭配无限制 | 20-40 天 |
| 顾客图鉴全解锁 | 接待所有类型顾客各 N 次 | 图鉴收集 100% | 15-25 天 |
| 完成全季节活动 | 经历春夏秋冬全部季节活动 | 季节限定全收集 | 30+ 天 |
| 完美咖啡大师 | 累计制作 500 杯五星咖啡 | 隐藏称号 | 20-30 天 |

| 超循环（季节） | 7-10天（游戏时间） | 季节更替→新内容 | 季节活动开启 |

## 2. 经济系统数值设计

### 2.1 设计原则

1. **无金币惩罚**：经营失败（顾客不满离开）只产生机会成本（少赚），不扣除金币；极度不满可能影响店铺口碑评价分，但不影响金币余额
2. **渐进增长**：金币产出随等级提升呈指数与线性混合增长
3. **稀缺感**：钻石始终稀缺，引导玩家完成高价值行为
4. **消费抉择**：金币用途多样（设备/家具/原料/工资），玩家需做优先级决策
5. **不肝不氪**：无内购设计，所有货币通过游戏行为获得

### 2.2 货币定义

| 货币 | 图标 | 定位 | 获取难度 | 主要用途 |
|------|------|------|---------|---------|
| 金币（Gold） | 🪙 | 通用货币 | 容易 | 设备升级、家具购买、原料采购、员工工资 |
| 钻石（Diamond） | 💎 | 稀有货币 | 较难 | 加速建造、稀有装饰、限定道具、跳过等待 |
| 好评积分（Reputation）| ⭐ | 口碑值 | 中 | 影响客流量、解锁特殊顾客 |

#### 2.3.3 金币产出速率公式

```
单顾客金币收入 = 商品基础售价 × 品质系数 × 顾客消费系数 × (1 + 店铺环境加成)

其中：
- 品质系数采用分段查找表（非连续线性的原因：鼓励玩家追求高星，4-5星回报递增明显）：
  | 1星 | 2星 | 3星 | 4星 | 5星 |
  |-----|-----|-----|-----|-----|
  | 0.7 | 0.8 | 0.9 | 1.2 | 1.5 |
- 顾客消费系数（见第3章顾客类型表）
- 店铺环境加成 = min(店铺环境分 / 500, 0.30)  （最高+30%）
```

### 2.4 钻石产出设计

钻石为稀缺货币，不通过常规经营大量产出，引导玩家关注成就和特殊事件。

| 获取途径 | 单次产出（钻石） | 频率/条件 | 日均预期产出 |
|---------|---------------|----------|------------|
| 每日签到 | 1-5（连续签到递增） | 每日 1 次 | 2-3 |
| 完成每日任务（全部） | 1-3 | 每日 | 2 |
| 顾客 5 星满意度 | 0-1（概率 10%） | 每次 5 星服务 | 0.5-2 |
| 特殊顾客任务 | 3-8 | 每周 2-3 次 | 1-3 |
| 成就解锁 | 5-50（按成就等级） | 一次性 | - |
| 季节活动 | 10-30 | 每季 | - |
| 随机事件奖励 | 2-5 | 每天 0-2 次 | 1-5 |
| 店铺升级 | 5-20 | 每次升级 | - |
| 咖啡大赛 | 10-50 | 每月 1 次 | 1-2 |

**钻石产出控制目标**：
- 新玩家首日：15-30 钻石（引导期多给）
- 日常稳定：5-15 钻石 / 天（游戏内）
- 活跃玩家月均：200-400 钻石

#### 2.5.2 设备升级费用公式

```
设备升级费用(n级 → n+1级) = 基础价格 × 升级系数^(n)

- 高级设备（Lv11-15）：系数 = 2.2
- 顶级设备（Lv16-20）：系数 = 2.5
```

### 3.2 顾客生成算法

#### 3.2.1 顾客生成流程

```
函数 generateCustomer(shopLevel, currentTime, environmentScore):
    1. 判断当前时段客流系数（见第8章时间系统）
    2. 判断是否触发特殊顾客（概率 = 0.05% × shopLevel × 客流系数）
    3. 根据 appearWeight 加权随机选择顾客类型
        - 筛选 appearTime 匹配当前时段的类型
        - 特殊顾客类型权重随 shopLevel ↑ 而 ↑
    4. 根据顾客类型创建顾客实例，赋值随机参数
    5. 从菜单中根据 preference 和 spendingPower 选择订单
    6. 返回顾客对象
```

#### 3.2.2 客流生成公式

```
每分钟新顾客数 = 基础客流 × 时段系数 × 店铺等级系数 × 环境加成系数 × 随机波动

其中：
- 基础客流 = 0.5 人/分钟（Lv1）
- 时段系数 = 见第8章顾客流量曲线
- 店铺等级系数 = 1 + (shopLevel - 1) × 0.08  （Lv1=1.0, Lv10=1.72, Lv20=2.52）
- 环境加成系数 = 1 + min(environmentScore / 200, 0.25)  （环境分0=1.0, 环境分50=1.25）
- 随机波动 = 0.8 ~ 1.2（均匀分布）

最大同时在店顾客数 = 座椅数 × 1.5（有人站着等）
```

### 3.3 满意度计算公式

#### 3.3.1 满意度计算模型

```
顾客满意度 = 基础满意度 + 品质分 + 环境分 + 速度分 - 等待惩罚

各项计算：

1. 基础满意度 = 顾客基础满意值（见3.1.2表）
   范围：60-90

2. 品质分 = (咖啡星级 - 3) × 10 × qualitySensitivity
   示例：5星咖啡 × 品质敏感度0.9 = +18分
        1星咖啡 × 品质敏感度0.9 = -18分

3. 环境分 = (店铺环境分 - 30) × 0.3 × environmentSensitivity
   示例：环境分80 × 敏感度0.8 = (80-30)×0.3×0.8 = +12分

4. 速度分 = max(0, (patience - 实际等待时间) / patience × 10 × (1 - speedSensitivity))
   示例：patience=60秒, 等了15秒, speedSensitivity=0.9
   = (60-15)/60 × 10 × 0.1 = +0.75分

5. 等待惩罚（超时前）：当等待时间超过 patience × 0.7 时开始生效
   等待惩罚 = (等待时间 / patience - 0.7) × 30 × speedSensitivity
   示例：等待54秒/60秒, speedSensitivity=0.9
   = (54/60 - 0.7) × 30 × 0.9 = 5.4分扣减

最终满意度 = clamp(满意度, 0, 100)
```

#### 3.3.2 满意度等级

| 满意度分数 | 星级评价 | 结果 | 额外效果 |
|-----------|---------|------|---------|
| 0-19 | ⭐ | 极不满意 | 顾客生气离开，店铺口碑-3（不影响金币）|
| 20-39 | ⭐⭐ | 不满意 | 正常付款，不会再来（短期内） |
| 40-59 | ⭐⭐⭐ | 一般 | 正常付款，中性评价 |
| 60-79 | ⭐⭐⭐⭐ | 满意 | 正常付款，可能成为回头客 |
| 80-95 | ⭐⭐⭐⭐⭐ | 非常满意 | 正常付款+小费概率触发，口头推荐+1客流 |
| 96-100 | 👑 | 完美体验 | 小费必定触发（1.5倍），发朋友圈推荐+3客流 |

### 3.4 小费公式

```
触发小费的条件: 满意度 ≥ 80 且 random(0,100) < 顾客tipRate × 100

小费金额 = 商品售价 × 小费系数

小费系数表：
- 满意度 80-89: 0.05-0.10
- 满意度 90-95: 0.10-0.20
- 满意度 96-100: 0.20-0.30

钻石小费（触发概率独立计算）：
- 满意度 96-100: 5% 概率获得 1 钻石
```

### 3.5 排队规则

```
排队算法（FIFO + 优先级微调）：

1. 顾客进入店铺 → 加入队列末尾
2. 如果有空位 → 直接入座
3. 如果满座 → 进入等待队列（最多排队人数 = 座位数 × 0.5）
4. 如果等待队列已满 → 顾客离开（"太挤了，下次再来"），不扣评价
5. 有空位时 → 队列头部顾客入座
6. 顾客入座后开始计时等待服务
7. 服务员（玩家/员工）依次接单

VIP优先：回头客（第5次+来访）可优先于普通顾客入座
预约顾客（特殊事件）：固定时间到达，保留座位
```

#### 3.5.1 排队耐心衰减

```
顾客在排队时耐心以 0.3× 速率衰减（排队比入座后等待的耐心消耗慢）
若排队时间超过 patience × 0.5，顾客离开
```

### 4.9 品质系统（1-5星）

#### 4.9.1 品质分计算模型

```
最终品质分 = 设备品质分 + 原料品质分 + 操作分 + 员工加成 - 随机波动

各分项详解：

1. 设备品质分（满分40分）
   = 相关设备品质加成总和 × 40
   示例：意式咖啡机品质加成30% → 12分

2. 原料品质分（满分30分）
   普通原料：每种 +3 分（上限15分）
   精品原料：每种 +5 分（上限25分）
   庄园级原料：每种 +8 分（上限30分）

3. 操作分（满分30分）
   = 玩家操作准确度 × 30
   - 每次步骤点击时机准确（绿色区域）= 1.0
   - 每次步骤点击时机一般（黄色区域）= 0.6
   - 每次步骤点击时机失误（红色区域）= 0.2
   - 拉花等额外步骤：完美+5分，一般+2分，无-0分

4. 员工加成（满分10分）
   = 员工品质加成 × 10
   咖啡师/甜品师每级提供额外品质分

5. 随机波动：±3分（模拟现实中的微小差异）

总分范围：0-110分
```

#### 4.9.2 星级判定

| 品质总分 | 星级 | 售价系数 | 顾客满意度品质分贡献 |
|---------|------|---------|-------------------|
| 0-24 | ⭐ | 0.7x | -20 |
| 25-44 | ⭐⭐ | 0.8x | -10 |
| 45-64 | ⭐⭐⭐ | 0.9x | 0 |
| 65-84 | ⭐⭐⭐⭐ | 1.2x | +12 |
| 85-100 | ⭐⭐⭐⭐⭐ | 1.5x | +25 |
| 101-110 | 👑 完美 | 1.8x | +35 |

> 完美判定需同时满足：品质分≥101、无步骤失误、使用至少一种精品以上原料

#### 4.9.3 品质稳定性

随着设备升级和员工成长，品质下限逐渐提升：

| 条件 | 品质下限 |

### 6.4 心情值机制

#### 6.4.1 心情值影响

```
心情值范围：0-100

心情值对工作效率的影响：
- 80-100：效率 +10%（愉快工作）
- 60-79：效率 ±0%（正常工作）
- 40-59：效率 -10%（有点疲惫）
- 20-39：效率 -20%（不想工作）
- 0-19：效率 -40%（极度不满），员工可能辞职

心情值对品质的影响：
- 品质分修正 = (心情值 - 50) × 0.1
  示例：心情 80 → 品质分+3，心情 30 → 品质分-2
```

#### 6.4.2 心情值变化规则

| 行为 | 心情值变化 | 备注 |
|------|----------|------|
| 每日工作（无休息） | -15/天 | 每日结算时扣除 |
| 发工资（正常日薪） | +10/天 | 每日工资发放恢复 |
| 发奖金（日薪×50%） | +30 | 每周限2次 |
| 安排休息（1天） | +25 | 休息日不工作，不领工资 |
| 店铺升级 | +10 | 一次性 |
| 制作出5星产品（咖啡师/甜品师） | +2 | 每次 |
| 获得顾客好评（服务员） | +2 | 每次 |
| 顾客不满离开 | -5 | 每个不满顾客 |
| 连续工作7天无休 | -10/天（额外） | 叠加扣减 |
| 节日/活动日 | +15 | 一次性 |
| 店猫互动 | +3 | 每天1次 |

#### 6.4.3 心情值应急机制

```
当员工心情值低于 20 时：
1. 弹出警告提示
2. 员工工作效率降低 40%
3. 连续 3 天心情值低于 20 → 员工辞职（永久失去）
4. 辞职后有 3 天冷却期才能招聘新员工
```

## 8. 时间系统设计

### 8.1 时间流速

游戏采用实时制 + 可选加速，设计为休闲节奏。

| 参数 | 数值 | 说明 |
|------|------|------|
| 游戏1天 = 现实时间 | 12 分钟 | 默认速度，可加速 |
| 加速模式 | 1x / 2x / 4x | 4x加速 = 游戏1天 = 3分钟 |
| 游戏1小时 = 现实时间 | 30 秒 | 1x 速度下 |
| 营业时间 | 06:00 - 22:00 | 共16个游戏小时 = 现实8分钟 |
| 闭店时间 | 22:00 - 06:00 | 共8个游戏小时 |

#### 8.1.1 加速模式

| 加速倍率 | 游戏1天耗时 | 解锁条件 | 适用场景 |
|---------|-----------|---------|---------|
| 1x | 12 分钟 | 默认 | 正常经营 |
| 2x | 6 分钟 | 店铺 Lv3 | 熟悉后加速 |
| 4x | 3 分钟 | 店铺 Lv8 | 后期快速过天 |

加速期间顾客行为自动加速，员工工作速度同步加速，确保游戏平衡。

#### 8.5.1 天气类型与参数

#### 8.5.2 天气切换规则

```
每日天气在 00:00（游戏时间）通过随机算法确定：

1. 获取当前季节天气概率分布
2. 加权随机选择基础天气
3. 特殊天气额外判定：
   - 前一天下雨 → 第二天彩虹概率 5%
   - 连续3天晴天 → 小雨概率+10%
   - 冬季连续5天晴天 → 小雪概率+15%
   - 冬季下雪 → 第二天大雪概率 20%
4. 天气在一天内不变（简化设计）
5. 特殊事件可临时改变天气（如「雨过天晴」事件）
```

## 附录 B：设计理念说明

### B.1 为什么品质系数从0.7起（而非0.5或1.0）

0.7的起点保证1星咖啡仍有价值（不会太亏），形成「做好→多赚」的激励而非「做差→惩罚」的挫败。1.5的5星上限提供50%的溢价空间，让追求品质有明确的收益回报。

### B.2 为什么钻石不设计为内购

本游戏定位为「不肝不氪」，钻石作为稀缺资源驱动玩家探索游戏的各个系统（成就、事件、任务），而非通过付费跳过内容。所有玩家在同一规则下成长，形成公平的社区氛围。

### B.3 顾客排队算法的友好设计

排队队列满时顾客离开「不扣评价」，只产生机会成本。这一设计遵循PRD中的「无惩罚设计」原则，让玩家专注于提升效率而非担心损失。

### B.4 数值的弹性空间

所有数值均基于表格计算，在代码中实现为可配置项。核心公式中的系数（如升级系数、品质系数、客流系数）都预留了 ±20% 的调整空间，便于内测期间根据玩家反馈快速调参。
